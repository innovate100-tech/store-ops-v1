# 이메일 인증 문제 해결 방법

회원가입 후 로그인 시 "이메일 인증이 필요합니다" 오류가 발생하는 경우 해결 방법입니다.

## 🔧 해결 방법 1: Supabase 대시보드에서 사용자 이메일 수동 확인 (권장)

### 단계별 가이드

1. **Supabase 대시보드 접속**
   - https://supabase.com 접속
   - 프로젝트 선택

2. **Authentication > Users 메뉴로 이동**
   - 왼쪽 메뉴에서 **Authentication** 클릭
   - **Users** 탭 선택

3. **사용자 찾기**
   - 회원가입한 이메일로 검색
   - 해당 사용자 행 클릭

4. **이메일 수동 확인**
   - 사용자 상세 화면에서
   - **"Confirm email"** 버튼 클릭 또는
   - **"Email Confirmed"** 체크박스 활성화

5. **로그인 재시도**
   - 앱에서 다시 로그인 시도
   - 정상적으로 로그인됩니다

---

## 🔧 해결 방법 2: Supabase 설정에서 이메일 인증 비활성화 (개발 환경용)

⚠️ **주의**: 이 방법은 개발 환경에서만 사용하세요. 프로덕션에서는 보안상 이메일 인증을 활성화하는 것이 좋습니다.

### 단계별 가이드

1. **Supabase 대시보드 접속**
   - https://supabase.com 접속
   - 프로젝트 선택

2. **Authentication > Settings 메뉴로 이동**
   - 왼쪽 메뉴에서 **Authentication** 클릭
   - **Settings** 탭 선택

3. **이메일 인증 설정 변경**
   - **"Enable Email Signup"**: ON 유지
   - **"Confirm email"**: **OFF**로 변경
   - **"Save"** 버튼 클릭

4. **기존 사용자 이메일 확인 (선택사항)**
   - 방법 1을 사용하여 기존 사용자의 이메일을 수동으로 확인

5. **로그인 재시도**
   - 앱에서 다시 로그인 시도
   - 정상적으로 로그인됩니다

---

## 📝 참고사항

- **개발 환경**: 이메일 인증을 비활성화하여 빠르게 테스트 가능
- **프로덕션 환경**: 보안을 위해 이메일 인증을 활성화하는 것이 권장됩니다
- **기존 사용자**: 방법 1을 사용하여 수동으로 이메일을 확인할 수 있습니다

---

## 🔧 해결 방법 3: 이메일 인증 후 리다이렉트 URL 설정 (선택사항)

인증 링크 클릭 후 공백 페이지가 뜨는 문제를 해결하려면:

1. **Supabase 대시보드 접속**
   - https://supabase.com 접속
   - 프로젝트 선택

2. **Authentication > URL Configuration 메뉴로 이동**
   - 왼쪽 메뉴에서 **Authentication** 클릭
   - **URL Configuration** 탭 선택

3. **Site URL 설정**
   - **Site URL**: 앱의 메인 URL 입력 (예: `https://your-app.streamlit.app` 또는 `http://localhost:8501`)
   - **Redirect URLs**: 추가 리다이렉트 URL 입력 (선택사항)
   - **Save** 버튼 클릭

4. **확인**
   - 다음 회원가입 시 이메일 인증 링크 클릭 후 설정한 URL로 리다이렉트됩니다

⚠️ **참고**: 
- 인증 링크 클릭 후 공백 페이지가 떠도 인증은 정상적으로 완료됩니다
- 앱에 다시 접속하면 인증된 상태로 로그인할 수 있습니다
- 이 설정은 사용자 경험 개선을 위한 선택사항입니다

---

## 🐛 문제가 계속되면

1. Supabase 대시보드에서 사용자 목록 확인
2. 사용자의 "Email Confirmed" 상태 확인
3. 필요시 사용자 삭제 후 재가입
