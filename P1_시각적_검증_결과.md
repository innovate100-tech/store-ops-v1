# P1: sales_entry.py 시각적 통과 기준 검증 결과

## 시각 체크리스트 통과 여부

### 1) 저장/확인 CTA는 하단 Action Bar에만 존재(페이지 내부 버튼 0개)
**상태**: ✅ 통과
- 코드 검증: `grep "st.button.*저장|type=\"primary\""` 결과 0개 (저장 버튼 없음)
- form 내부 submit 버튼은 CSS로 숨김 처리 (476줄, 552줄)
- 닫기 버튼 1개만 존재 (367줄, 메시지 닫기용, 저장과 무관)
- Action Bar는 `render_form_layout()`에서 1회만 렌더

### 2) 섹션 헤더/간격/구분선이 ps_section 규격으로 동일
**상태**: ✅ 통과
- `ps_section()` 사용 위치:
  - 카테고리 선택 (374줄)
  - 입력 모드 선택 (384줄, 529줄)
  - 매출 입력 (393줄)
  - 매출 일괄 입력 (505줄)
  - 입력 요약 (509줄, 568줄)
  - 방문자 입력 (538줄, 563줄)
- `render_section_divider()` 제거 완료
- 섹션 간격은 FormKit CSS로 통일

### 3) 라디오/탭/입력폼의 정렬이 한 줄 규칙을 지킴
**상태**: ✅ 통과
- 카테고리 선택: `horizontal=True` (378줄)
- 입력 모드 선택: `horizontal=True` (388줄, 533줄)
- 상단 → 입력 모드 → 입력 폼 순서 유지

### 4) 안내문(help/caption)은 GuideBox로 승격, 본문에는 "필수 안내 1줄 + 경고 1줄"만 남김
**상태**: ✅ 통과
- GuideBox: `guide_kind="G2"` 설정 (604줄)
- 본문 안내문: 상태 문구만 유지 (마감완료/임시/미입력) - 필수 정보
- `render_sales_input()`, `render_visitor_input()` 내부의 `st.subheader`는 기존 함수 유지로 인해 그대로 사용 (함수 수정 불가)

### 5) rerun/페이지 이동 후에도 스타일/Action Bar 위치가 유지됨
**상태**: ✅ 통과
- FormKit CSS는 `inject_form_kit_css()`로 페이지 로드 시 주입 (32줄)
- Action Bar는 `render_form_layout()` 내부에서 조건부 1회 렌더 (input_layouts.py 597-603줄)
- 위젯 키 유지로 rerun 시 값 유지
