## 문제점 분석

### 1. **명명 불일치**
- "메뉴 포트폴리오 설계" → 실제 페이지는 "메뉴 등록" (설계 vs 등록)
- "재료 구조 설계" → 실제 페이지는 "재료 등록" (설계 vs 등록)
- "레시피 설계" → 실제 페이지는 "레시피 등록" (설계 vs 등록)
- "원가 분석" → 실제 페이지는 "비용 분석" (원가 vs 비용)
- "QSC 입력" → 실제 페이지는 "건강검진 실시" (QSC vs 건강검진)

### 2. **카테고리 분류 모호**
- "설계" 카테고리가 실제로는 "등록" 기능 (설계 vs 등록 혼재)
- "분석" 카테고리에 "체크 결과 요약", "체크 히스토리" 포함 (분석 vs 검진)
- "입력" 카테고리에 "목표" 입력 포함 (목표 설정은 입력이 아닌 설계)

### 3. **중요 기능 누락**
- **재료 사용량 집계**: 발주 관리의 핵심 기능인데 사이드바에 없음
- **발주 관리**: 재고 관리의 핵심 기능인데 사이드바에 없음
- **재고 분석**: 재고 관리의 분석 기능인데 사이드바에 없음
- **주간 리포트**: 성과 리포트인데 사이드바에 없음
- **오늘의 전략 실행**: 전략 실행 기능인데 사이드바에 없음

### 4. **사용 빈도 고려 부족**
- 일일 마감 입력은 가장 자주 사용하는데 서브 메뉴에 숨어있음
- 매출 등록도 자주 사용하는데 서브 메뉴에 숨어있음
- 분석 허브는 허브 역할인데 서브 메뉴가 많아 접근성이 떨어짐

### 5. **용어 통일성 부족**
- "설계" vs "등록" vs "입력" 혼재
- "분석" vs "관리" vs "파악" 혼재
- "QSC" vs "건강검진" vs "체크" 혼재

## 개선 방향

### 1. **명확한 카테고리 분류**
- **일일 운영**: 매일 사용하는 기능 (마감, 입력)
- **설계/설정**: 초기 설정 및 목표 설정
- **분석/리포트**: 데이터 분석 및 리포트
- **운영 지원**: 연락망, 게시판 등

### 2. **사용 빈도 기반 배치**
- **최상단**: 홈, 일일 마감 (가장 자주 사용)
- **상단**: 매출 입력, 판매량 입력 (자주 사용)
- **중단**: 분석, 리포트 (주기적 사용)
- **하단**: 설계, 설정 (초기/주기적 설정)

### 3. **명명 통일**
- **등록/입력**: 데이터 입력 기능 → "입력"으로 통일
- **설계/설정**: 초기 설정 및 목표 → "설정"으로 통일
- **분석/관리**: 데이터 분석 → "분석"으로 통일
- **QSC/건강검진**: "품질 검사" 또는 "QSC 검사"로 통일

### 4. **누락 기능 추가**
- 재료 사용량 집계
- 발주 관리
- 재고 분석
- 주간 리포트
- 오늘의 전략 실행

```

   └─ 홈

📅 일일 운영
   └─ 오늘 마감 입력

✍ 데이터 입력
   └─ 데이터 입력 센터
      ├─ 매출/방문자 입력

      └─ QSC 검사 입력

   └─ 분석 허브

      ├─ 원가 분석
      ├─ 재고 분석

      ├─ 매출 하락 원인 찾기
      └─ 주간 리포트

   └─ 가게 전략 센터
      ├─ 오늘의 전략 실행
```

3. **재고 분석** → 분석 카테고리에 추가
4. **주간 리포트** → 새로운 "리포트" 카테고리 생성 또는 분석 카테고리에 추가

6. **분석 총평** → 분석 카테고리에 추가

### Phase 3: 카테고리 재구조화 (선택적)

- 일일 마감을 최상단으로 이동

- 서브 메뉴 최소화 (접근성 향상)
