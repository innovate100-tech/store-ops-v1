# 히어로 박스 멘트 개선안

## 🎯 목표
CAUSE OS 브랜드 히어로 박스의 멘트를 더욱 강력하고 명확하게 개선하여 브랜드 정체성과 가치 제안을 효과적으로 전달

---

## 📊 현재 멘트 분석

### 현재 구조
```
1. CAUSE OS (변경 불가)
2. 우리는 매출을 보지 않습니다.
   원인을 봅니다. (슬로건)
3. 사장을 위한 숫자 운영체제 (보조 정의)
```

### 현재 멘트의 특징
- ✅ **명확한 차별화**: "매출이 아닌 원인"이라는 핵심 가치
- ✅ **직설적**: 간결하고 이해하기 쉬움
- ⚠️ **개선 여지**: 감성적 연결, 행동 유도, 구체성

---

## 🔄 개선 제안

### [안 1] 현재 멘트 강화형 (추천 ⭐⭐⭐⭐⭐)

**슬로건:**
```
우리는 매출을 보지 않습니다.
원인을 봅니다.
```

**보조 정의:**
```
사장을 위한 숫자 운영체제
```

**변형 A:**
```
사장을 위한 데이터 운영체제
```

**변형 B:**
```
매장 운영의 원인을 찾는 시스템
```

**특징:**
- 현재 멘트의 강점 유지
- 약간의 표현만 조정
- 브랜드 정체성 일관성 유지

---

### [안 2] 가치 중심형

**슬로건:**
```
숫자 뒤에 숨은 원인을 찾습니다.
그래서 매장이 달라집니다.
```

**보조 정의:**
```
사장을 위한 숫자 운영체제
```

**변형 A:**
```
숫자가 말하는 진짜 원인을 봅니다.
그래서 결정이 달라집니다.
```

**변형 B:**
```
매출이 아니라 원인을 봅니다.
그래서 결과가 달라집니다.
```

**특징:**
- 결과/변화 강조
- 감성적 연결 강화
- "달라집니다"로 기대감 조성

---

### [안 3] 행동 유도형

**슬로건:**
```
우리는 매출을 보지 않습니다.
원인을 봅니다.
```

**보조 정의:**
```
입력 → 분석 → 설계, 3단계로 매장을 운영합니다
```

**변형 A:**
```
3단계로 매장을 데이터 기반으로 운영합니다
```

**변형 B:**
```
입력, 분석, 설계로 매장을 시스템화합니다
```

**특징:**
- 구체적인 방법론 제시
- 사용자 행동 가이드
- 기능 중심

---

- 성장/변화 강조

## 🎨 멘트 구성 원칙

### 1. 브랜드 정체성
- "원인을 본다"는 핵심 가치 유지
- CAUSE OS의 철학 반영

### 2. 명확성
- 한눈에 이해 가능
- 전문 용어 최소화

### 3. 감성적 연결
- 사장의 고민에 공감
- 해결책에 대한 기대감

### 4. 행동 유도
- 다음 행동이 명확
- 사용 동기 부여
