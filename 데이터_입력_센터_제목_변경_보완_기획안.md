# 데이터 입력 센터 제목 변경 보완 기획안

## 1. 문제 상황
데이터 입력 센터 페이지에서 일부 제목이 여전히 변경되지 않은 상태로 표시되고 있습니다.

## 2. 확인된 변경되지 않은 제목 목록

사용자가 확인한 변경되지 않은 제목들:
1. ✍ 입력 허브
2. 📊 실시간 입력 현황
3. 🏗️ 가게 데이터 기초 체력
4. ⚡ 매일 · 매주 · 매월 루틴
5. 🎯 목표 및 분석 기준
6. 🛠️ 가게 정의 (기초 뼈대)
7. ⚙️ 데이터 보정 도구 (과거 일괄 수정)

## 3. 변경 목표

모든 제목을 입력 중심으로 통일:

| 현재 제목 | 변경 후 제목 | 위치 | 변경 이유 |
|---------|------------|------|---------|
| ✍ 입력 허브 | ✍ 데이터 입력 센터 | 페이지 헤더 | 입력 중심의 명확한 정체성 강화 |
| 📊 실시간 입력 현황 | 📊 오늘의 입력 현황 | 섹션 제목 (###) | 입력 중심으로 재작성 |
| 🏗️ 가게 데이터 기초 체력 | 🏗️ 입력 데이터 완성도 | 섹션 제목 (###) | 입력 중심으로 재작성 |
| ⚡ 매일 · 매주 · 매월 루틴 | ⚡ 정기 입력 작업 | 섹션 제목 (####) | 입력 중심으로 재작성 |
| 🎯 목표 및 분석 기준 | 🎯 목표 입력 | 섹션 제목 (####) | 입력 중심으로 재작성 |
| 🛠️ 가게 정의 (기초 뼈대) | 🛠️ 기초 데이터 입력 | 섹션 제목 (####) | 입력 중심으로 재작성 |
| ⚙️ 데이터 보정 도구 (과거 일괄 수정) | ⚙️ 과거 데이터 입력 | Expander 제목 | 입력 중심으로 재작성 |

## 4. 파일별 변경 내역

### 4.1 `ui_pages/input/input_hub.py`

#### 변경 사항 1: 파일 상단 주석
```python
# 현재
"""
입력 허브 페이지
입력 관련 모든 페이지로의 네비게이션 허브 (4단계 고도화 버전 - 통합 가이드 및 워크플로우)
"""

# 변경 후
"""
데이터 입력 센터 페이지
입력 관련 모든 페이지로의 네비게이션 허브 (4단계 고도화 버전 - 통합 가이드 및 워크플로우)
"""
```

#### 변경 사항 2: 페이지 헤더 (확인 필요)
```python
# 현재 (이미 변경되어야 함)
render_page_header("✍ 입력 허브", "✍")

# 변경 후
render_page_header("✍ 데이터 입력 센터", "✍")
```

#### 변경 사항 3: 섹션 제목들 (확인 필요)
```python
# 현재 → 변경 후
st.markdown("### 📊 실시간 입력 현황") 
→ st.markdown("### 📊 오늘의 입력 현황")

st.markdown("### 🏗️ 가게 데이터 기초 체력")
→ st.markdown("### 🏗️ 입력 데이터 완성도")

st.markdown("#### ⚡ 매일 · 매주 · 매월 루틴")
→ st.markdown("#### ⚡ 정기 입력 작업")

st.markdown("#### 🎯 목표 및 분석 기준")
→ st.markdown("#### 🎯 목표 입력")

st.markdown("#### 🛠️ 가게 정의 (기초 뼈대)")
→ st.markdown("#### 🛠️ 기초 데이터 입력")

st.expander("⚙️ 데이터 보정 도구 (과거 일괄 수정)")
→ st.expander("⚙️ 과거 데이터 입력")
```

## 6. 가능한 원인 분석

### 6.1 브라우저 캐시 문제
- Streamlit 앱이 캐시된 버전을 표시할 수 있음
- 해결: 브라우저 새로고침 (Ctrl+F5) 또는 앱 재시작

### 2. 파일 저장 문제
- 변경 사항이 파일에 저장되지 않았을 수 있음
- 해결: 파일 저장 상태 확인

### 3. 다른 파일 참조
- 다른 위치에서 같은 함수를 호출하고 있을 수 있음
- 해결: 전체 프로젝트에서 "입력 허브" 검색

### 4. 코드 변경 누락
- 일부 위치의 제목이 변경되지 않았을 수 있음
- 해결: 파일 전체 검색 및 일괄 변경
