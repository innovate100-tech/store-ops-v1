# 이메일 인증 비활성화 가이드 (테스트용)

테스트 편의를 위해 Supabase에서 이메일 인증을 비활성화하는 방법입니다.

## 🔧 Supabase 대시보드에서 이메일 인증 비활성화

### 방법 1: Settings 메뉴에서 (권장)

1. **Supabase 대시보드 접속**
   - https://supabase.com 접속
   - 프로젝트 선택

2. **Settings 메뉴로 이동**
   - 왼쪽 하단의 **⚙️ Settings** (톱니바퀴 아이콘) 클릭
   - **Authentication** 메뉴 선택

3. **이메일 인증 설정 변경**
   - **"Enable Email Signup"**: ON 유지
   - **"Confirm email"**: **OFF**로 변경
   - **"Save"** 버튼 클릭

4. **확인**
   - 설정 저장 후 회원가입 시 이메일 인증 없이 바로 로그인 가능합니다

---

### 방법 2: Authentication 메뉴에서

1. **Supabase 대시보드 접속**
   - https://supabase.com 접속
   - 프로젝트 선택

2. **Authentication 메뉴로 이동**
   - 왼쪽 메뉴에서 **Authentication** 클릭

3. **Providers 탭에서 Email 설정 확인**
   - **Providers** 탭 선택
   - **Email** 프로바이더 확인
   - Email 프로바이더 옆의 **⚙️ 설정 아이콘** 또는 **"Configure"** 버튼 클릭

4. **이메일 인증 설정 변경**
   - **"Confirm email"** 또는 **"Email confirmation required"** 옵션을 **OFF**로 변경
   - **"Save"** 버튼 클릭

---

### 방법 3: 직접 URL 접근

1. Supabase 대시보드에서 프로젝트 선택
2. 브라우저 주소창에 다음 경로 추가:
   ```
   /project/[프로젝트-id]/auth/providers
   ```
   또는
   ```
   /project/[프로젝트-id]/settings/auth
   ```
3. Email 프로바이더 설정에서 "Confirm email" 옵션을 OFF로 변경

---

## ⚠️ 주의사항

- **개발 환경에서만 사용**: 이 설정은 개발/테스트 환경에서만 사용하세요
- **프로덕션 환경**: 보안을 위해 프로덕션에서는 이메일 인증을 활성화하는 것이 권장됩니다
- **기존 사용자**: 이미 가입한 사용자의 이메일 인증 상태는 변경되지 않습니다

---

## 🔄 설정 변경 후

1. 앱을 새로고침하거나 재시작
2. 새로 회원가입 시도
3. 이메일 인증 없이 바로 로그인 가능한지 확인

---

## 📝 참고

- 이 설정을 변경하면 **새로 가입하는 사용자**부터 적용됩니다
- 이미 가입한 사용자는 Supabase 대시보드에서 수동으로 이메일을 확인해야 할 수 있습니다
