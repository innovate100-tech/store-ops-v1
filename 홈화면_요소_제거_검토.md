# 홈화면 요소 제거 검토: 상태 요약 + 추천 블록

## 📋 제거 대상 요소

1. **"📍 오늘 당신의 매장은"** - 상태 요약 (미니 카드 3개)
   - 마감 상태 (✅/❌ / 5/7)
   - 레시피 커버율 (75%)
   - 목표 설정 상태 (✅/❌ / ✅/❌)

2. **"🎯 오늘 우리 매장 추천"** - 추천 메시지 블록
   - 추천 엔진 메시지
   - 액션 버튼
   - 최근 7일 코치 기록

---

## ✅ 제거 시 장점

### 1. 간결성 향상
- **섹션 수 감소**: 3개 → 1개 (브랜드 히어로만)
- **카드 수 감소**: 4개 → 1개 (히어로만)
- **정보 밀도 감소**: 핵심 메시지에 집중

### 2. UX 개선
- **"어렵다"는 느낌 해소**: 정보 과부하 제거
- **시각적 피로감 감소**: 카드 수 최소화
- **스크롤 불필요**: 한 화면에 모든 핵심 정보

### 3. 브랜드 정체성 강화
- **히어로 영역 강조**: CAUSE OS 브랜드가 더 돋보임
- **STEP 가이드 강조**: 3단계 흐름이 더 명확해짐
- **"운영 OS" 느낌**: 기능 대시보드가 아닌 브랜드 랜딩 페이지

---

## ⚠️ 제거 시 단점 및 고려사항

### 1. 상태 요약 제거 시

**기능적 손실:**
- 사용자가 현재 매장 상태를 한눈에 파악하기 어려움
- 마감 완료율, 레시피 커버율, 목표 설정 여부를 확인하려면 각 페이지로 이동 필요

**대안:**
- 상태 정보는 각 해당 페이지(입력/분석/설계)에서 확인 가능
- 홈은 "시작점" 역할에 집중

**판단:**
- ✅ **제거 가능**: 상태는 각 기능 페이지에서 확인하는 것이 더 자연스러움
- 홈은 "어디로 가야 할지"를 알려주는 역할에 집중

---

### 2. 추천 블록 제거 시

**기능적 손실:**
- 개인화된 가이드 제공 기능 상실
- 추천 엔진 로깅 데이터 수집 불가
- 사용자가 "오늘 무엇을 해야 할지"에 대한 명확한 가이드 부재

**기술적 고려:**
- `get_home_recommendation_v1()` 함수는 여전히 존재하지만 사용되지 않음
- `log_reco_event()` 로깅 기능도 사용되지 않음
- 추천 엔진 로직은 유지되지만 UI에서 제거

**대안:**
- 추천 기능은 각 기능 페이지에서 컨텍스트에 맞게 제공 가능
- 예: 입력 페이지에서 "어제 마감이 안 되어 있습니다" 같은 안내

**판단:**
- ⚠️ **조건부 제거 가능**: 
  - 장점: 홈화면 간결성 확보
  - 단점: 개인화 가이드 기능 상실
  - **결론**: 사용자 피드백("어렵다")을 우선시한다면 제거 검토 가능

## 🎯 제거 후 홈화면 구조

### 최종 구조 (제거 후)

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
│  - CAUSE OS                         │
│  - 슬로건                           │
│  - [오늘 숫자 입력하기] CTA         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 가이드 (압축 또는 확장)        │
│  - STEP 1: 입력                     │
│  - STEP 2: 분석                     │
│  - STEP 3: 설계                     │
└─────────────────────────────────────┘
```

**특징:**
- 2개 섹션만 (히어로 + STEP)
- 브랜드 정체성과 전체 흐름에 집중
- "어디로 가야 할지" 명확함

## 💡 하이브리드 옵션 (부분 제거)

### 옵션 A: 상태 요약만 제거
- 상태 미니 카드 3개 제거
- 추천 블록은 유지
- **효과**: 정보 밀도 감소, 추천 기능 유지

### 옵션 B: 추천 블록만 제거
- 추천 메시지 + 버튼 제거
- 상태 요약은 유지
- **효과**: 개인화 가이드는 제거하되 상태 정보는 유지

### 옵션 C: 모두 제거 (추천 ⭐⭐⭐⭐⭐)
- 상태 요약 + 추천 블록 모두 제거
- **효과**: 최대 간결성, 브랜드 + STEP에 집중

## 📊 기능적 영향 분석

### 제거해도 문제없는 이유

1. **상태 정보는 각 페이지에서 확인 가능**
   - 마감 상태 → 일일 입력 페이지
   - 레시피 커버율 → 메뉴/레시피 페이지
   - 목표 설정 → 목표 페이지

2. **추천 기능은 컨텍스트에서 제공 가능**
   - 입력 페이지: "어제 마감이 안 되어 있습니다"
   - 분석 페이지: "분석 가능한 데이터가 부족합니다"
   - 설계 페이지: "목표를 먼저 설정하세요"

3. **홈의 역할 재정의**
   - 현재: "상태 대시보드 + 추천 + 가이드"
   - 변경 후: "브랜드 정체성 + 전체 흐름 가이드"
