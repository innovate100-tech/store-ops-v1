# 홈화면 맥락 추가 방안: 최대 간결형 + 앱 설명

## 🎯 목표
- 최대 간결형 유지 (히어로 + STEP만)
- 앱 설명과 활용 맥락 추가
- "어렵다"는 느낌 해소하면서 이해도 향상

---

## 💡 방법안

### [방법 1] 히어로 아래 짧은 설명 블록 추가 (추천 ⭐⭐⭐⭐⭐)

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
│  - CAUSE OS                         │
│  - 슬로건                           │
│  - [오늘 숫자 입력하기] CTA         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  이 앱이 하는 일 (간단 설명)        │
│  - 입력 → 분석 → 설계 3단계         │
│  - 데이터 자산으로 매장 운영        │
│  - 원가율 5% 개선 = 월 100만원 이익 │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 가이드                        │
│  [입력] [분석] [설계]               │
└─────────────────────────────────────┘
```

**특징:**
- 얇은 설명 카드 (높이 작게)
- 핵심 가치 제안만 (2-3줄)
- 브랜드와 STEP 사이 연결고리 역할

---

### [방법 2] STEP 카드 확장 (설명 포함)

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 1: 입력                       │
│  [간단 설명]                        │
│  [버튼]                             │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 2: 분석                       │
│  [간단 설명]                        │
│  [버튼]                             │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 3: 설계                       │
│  [간단 설명]                        │
│  [버튼]                             │
└─────────────────────────────────────┘
```

**특징:**
- 각 STEP에 1-2줄 설명 추가
- 카드 수는 3개 유지
- 각 STEP의 가치 명확히

---

### [방법 3] STEP 카드에 Expander로 상세 설명

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 1: 입력 [버튼]               │
│  [▼ 자세히 보기] (expander)         │
│    - 설명 펼침                      │
└─────────────────────────────────────┘
```

**특징:**
- 기본은 간결하게
- 필요시 펼쳐서 상세 설명
- 선택적 정보 접근

---

### [방법 4] 히어로와 STEP 사이 가이드 텍스트

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
└─────────────────────────────────────┘

[작은 가이드 텍스트]
"이 앱은 입력 → 분석 → 설계 3단계로
매장을 데이터 기반으로 운영하게 만듭니다.
레시피만 등록하면 원가율이 자동 계산되고,
원가율 5% 개선으로 월 100만원 추가 이익을
만들 수 있습니다."

┌─────────────────────────────────────┐
│  STEP 가이드                        │
└─────────────────────────────────────┘
```

**특징:**
- 카드 없이 텍스트만
- 얇고 연한 톤
- 맥락 제공

## 🎨 추천안: [방법 1] + [방법 2] 하이브리드

### 구조

```
┌─────────────────────────────────────┐
│  브랜드 히어로                       │
│  - CAUSE OS                         │
│  - 슬로건                           │
│  - [오늘 숫자 입력하기] CTA         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  이 앱이 하는 일 (얇은 설명 블록)   │
│  ┌───────────────────────────────┐  │
│  │ 입력 → 분석 → 설계 3단계로    │  │
│  │ 매장을 데이터 기반 운영        │  │
│  │ 원가율 5% 개선 = 월 100만원   │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 1: 입력                      │
│  "매장을 데이터 자산으로 만든다"    │
│  - 메뉴/재료/레시피/매출/비용 입력  │
│  [▶ 입력하기] [▶ 입력센터]         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 2: 분석                      │
│  "숫자가 문제를 말해준다"           │
│  - 매출 원인 / 새는 곳 / 키울 것    │
│  [▶ 분석하기]                      │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  STEP 3: 설계                      │
│  "숫자를 행동으로 바꾼다"           │
│  - 개선 우선순위 / 전략 / 방향      │
│  [▶ 설계하기]                      │
└─────────────────────────────────────┘
```

**특징:**
- 설명 블록: 얇고 작은 폰트, 연한 톤
- STEP 카드: 각각 간단한 설명 포함
- 총 4개 섹션 (히어로 + 설명 + STEP 3개)

## ✅ 최종 추천 구조

### 섹션 구성
1. **브랜드 히어로** (기존 유지)
2. **앱 가이드** (새로 추가 - 얇은 설명 블록)
3. **STEP 1-3** (각각 설명 포함)

### 디자인 원칙
- 설명 블록: 높이 얇게, 작은 폰트, 연한 톤
- STEP 카드: 각각 1-2줄 설명 포함
- 전체: 여전히 간결하되 맥락 제공

- "어렵다"는 느낌 해소

**구조:**
- 4개 섹션 (히어로 + 가이드 + STEP 3개)
- 각 섹션 역할 명확
