# ✍️ 입력 허브 고도화 기획안 (Upgrade Plan)

## 1. 개요 (Overview)
입력 허브는 사용자가 가게의 데이터를 시스템에 주입하는 **'현실 반영의 입구'**입니다. 현재의 단순 리스트 형태를 넘어, **입력의 완결성과 자산 구축의 진척도**를 한눈에 파악할 수 있는 **'입력 관제 센터(Input Control Center)'**로 진화합니다.

## 2. 주요 개선 방향 (Key Improvements)
- **현황 중심의 요약**: 단순 링크가 아닌, 현재 무엇이 입력되었고 무엇이 누락되었는지 시각적 요약(카드) 제공.
- **데이터 구축 진척도(Asset Readiness)**: 메뉴, 재료, 레시피 등 가게의 기초 자산이 얼마나 완성되었는지 퍼센트(%) 또는 개수로 표시.
- **주기별 명확한 분리**: 매일(Daily), 정기(Regular), 기초(Base) 작업을 사용자의 인지 흐름에 맞춰 재배치.
- **UX 가이드 강화**: 다음에 해야 할 작업을 명확히 지시하는 추천 로직 고도화.

---

## 3. UI/UX 상세 설계 (Detailed UI/UX Design)

### [ZONE A] 📊 입력 관제 보드 (Control Board)
- **구성**: 3개의 핵심 상태 카드
  1. **오늘의 마감**: [완료/미완료] + (매출액/방문자수 요약 표시)
  2. **이번 주 QSC**: [N회 완료] + (최근 실시일 표시)
  3. **이번 달 정산**: [완료/미완료] + (정산된 이익금액 요약 표시)

### [ZONE B] 🏗️ 가게 자산 구축 현황 (Asset Readiness)
- 사용자가 시스템을 제대로 쓰기 위해 갖춰야 할 기초 데이터의 완성도를 보여줌.
- **구성**:
  - **메뉴 정보**: 등록된 메뉴 개수 (예: 24개)
  - **재료 정보**: 등록된 재료 개수 (예: 85개)
  - **레시피 완성도**: 레시피가 등록된 메뉴 비율 (예: 75%)
  - **목표 설정**: 매출/비용 목표 설정 여부 (✅/⚠️)

### [ZONE C] ⚡ 작업 영역 (Task Areas)
작업 빈도와 성격에 따라 3개의 탭 또는 섹션으로 구분.

#### 1. 일일 & 정기 작업 (Routine)
- **일일 마감 입력**: 가장 빈번한 작업. 눈에 띄는 기본 버튼.
- **QSC 점검 / 월간 실제 정산**: 주기적 작업. 상태 표시와 함께 배치.

#### 2. 가게 기초 정의 (Foundation)
- 메뉴 관리, 재료 관리, 레시피 관리, 재고 관리.
- '이 앱의 뼈대를 만드는 곳'이라는 설명과 함께 그룹화.

#### 3. 기준 설정 & 보정 (Config & Admin)
- 목표 매출/비용 구조 설정.
- 과거 데이터 보정 (매출/판매량 일괄 등록).

---

## 4. 데이터 로직 고도화 (Data Logic)
- **실시간 카운팅**: `menu_master`, `ingredients`, `recipes` 테이블에서 개수를 실시간 count하여 현황판에 반영.
- **상태 판별**: 오늘 날짜의 `daily_close` 존재 여부, 최근 7일 내 `health_check_sessions` 완료 여부 등을 체크하여 카드 색상 변경 (초록/노랑).

## 5. 기대 효과 (Expected Benefits)
- **입력 누락 방지**: 무엇을 안 했는지 허브에서 즉시 확인 가능하여 데이터 연속성 확보.
- **사용자 성취감**: 자산 구축 개수가 늘어나는 것을 보며 시스템 안착에 대한 동기부여 제공.
- **명확한 네비게이션**: 복잡한 입력 페이지들 사이에서 길을 잃지 않고 현재 필요한 작업으로 즉시 이동.
