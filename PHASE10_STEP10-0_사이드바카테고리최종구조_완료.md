## 주요 변경 사항

### 1. 카테고리 재구성
- **기존**: "🟡 오늘 가게", "🟢 오늘 입력", "🔵 내일 준비", "🟣 장사 분석", "🔴 성적표", "🔥 가게 설계"
- **변경**: "🏠 HOME", "🔥 가게 설계", "🚨 문제 분석", "🛠 운영 입력 · 관리", "📄 성적표 · 리포트", "👥 운영 도구"

### 2. "🔥 가게 설계" 최상단 고정
- 사이드바 렌더링 로직 수정: "🔥 가게 설계"를 먼저 렌더링하도록 변경
- 가게 설계 센터가 최상단에 위치하도록 보장

### 3. 직원/협력사/게시판 분리
- **기존**: "🔥 가게 설계" 카테고리에 포함
- **변경**: "👥 운영 도구" 카테고리로 이동

### 4. 설계실 정리
- "🔥 가게 설계" 카테고리에서 설계실 4개만 유지
- "목표 비용 구조", "목표 매출 구조"는 "🛠 운영 입력 · 관리"로 이동

### 5. 문제 분석 카테고리 신설
- "🚨 문제 분석" 카테고리 추가
- 매출 분석, 판매·메뉴 분석, 원가 분석 포함
- 향후 "매출 하락 원인 찾기", "방문·객단가 분석" 추가 예정

## 주의사항

1. **중복 page key**: 
   - "메뉴 관리"와 "메뉴 포트폴리오 설계실"이 같은 page key("메뉴 등록")를 사용
   - "재료 관리"와 "재료 구조 설계실"이 같은 page key("재료 등록")를 사용
   - 라벨만 다르게 표시되며, 실제 라우팅은 동일한 페이지로 이동

2. **향후 추가 예정 페이지**:
   - "매출 하락 원인 찾기" (주석 처리됨)
   - "방문·객단가 분석" (주석 처리됨)
   - 필요 시 주석을 해제하고 해당 페이지를 구현하면 됨

3. **"🔥 가게 설계" expander**:
   - 현재 expander로 표시되며, 최상단에 고정됨
   - 필요 시 일반 카테고리로 변경 가능

## 검증 방법

1. 앱 실행 후 사이드바 확인
2. "🔥 가게 설계"가 최상단에 위치하는지 확인
3. 가게 설계 센터가 첫 번째 항목인지 확인
4. 직원/협력사/게시판이 "👥 운영 도구"에 있는지 확인
5. 모든 페이지 라우팅이 정상 작동하는지 확인
