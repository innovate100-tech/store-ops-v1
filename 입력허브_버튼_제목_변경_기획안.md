# 데이터 입력 센터 페이지 개선 기획안

## 1. 개요
입력허브 페이지를 **"데이터 입력 센터"**로 명칭 변경하고, 페이지 전체를 **"입력"**이라는 단어가 중심이 되도록 재구성합니다. 페이지 제목, 섹션 제목, 버튼 제목을 모두 입력 중심으로 통일하여 사용자가 입력 작업임을 명확히 인지할 수 있도록 합니다.

## 2. 변경 원칙

### 2.1 핵심 원칙
- **페이지 명칭 변경**: "입력 허브" → **"데이터 입력 센터"**
- **"입력" 중심 구성**: 페이지 제목, 섹션 제목, 버튼 제목 모두 입력 중심으로 재구성
- **명확성 강화**: 모든 요소에서 입력 작업임을 즉시 인지 가능
- **일관성 유지**: 동일한 패턴으로 전체 페이지 구성

### 2.2 변경 대상 구분
- **페이지 제목**: "입력 허브" → "데이터 입력 센터"
- **섹션 제목**: 모든 섹션 제목을 입력 중심으로 재작성
- **입력 페이지 버튼**: 실제 데이터를 입력하는 페이지로 이동 → "입력"으로 끝남
- **설정/관리 페이지 버튼**: 설정이나 관리 페이지로 이동 → 기존 명칭 유지 또는 "설정"으로 끝남

## 5. UI/UX 개선 효과

### 5.1 명확성 향상
- 모든 입력 버튼이 "입력"으로 끝나 사용자가 즉시 입력 작업임을 인지
- 버튼 제목만으로도 페이지의 성격을 파악 가능

### 5.2 일관성 확보
- 동일한 패턴으로 버튼 제목 구성하여 학습 부담 감소
- "입력"이라는 키워드로 입력 허브의 정체성 강화

### 5.3 사용자 경험 개선
- "입력"이라는 단어가 중심이 되어 입력 허브의 목적이 명확해짐
- 버튼 클릭 전에 예상되는 동작을 쉽게 파악 가능

## 6. 구현 시 주의사항

### 6.1 데이터 입력 센터 페이지 제목 변경
- `ui_pages/input/input_hub.py` 파일의 `render_page_header("✍ 입력 허브", "✍")` → `render_page_header("✍ 데이터 입력 센터", "✍")`로 변경

### 6.2 사이드바 메뉴 변경
- `app.py` 파일의 사이드바 메뉴 딕셔너리에서 라벨 변경
- 페이지 키(`key`)는 변경하지 않음 (라우팅 호환성 유지)
- 예: `("입력 허브", "입력 허브")` → `("데이터 입력 센터", "입력 허브")`

### 6.3 섹션 제목 변경
- `ui_pages/input/input_hub.py` 파일의 모든 섹션 제목(`st.markdown("### ...")`)을 입력 중심으로 재작성
- 섹션 설명(`st.caption()`)도 입력 중심으로 조정

### 6.4 버튼 제목 변경
- `ui_pages/input/input_hub.py` 파일의 버튼 라벨만 변경
- 페이지 키(`st.session_state.current_page`)는 변경하지 않음

### 6.5 연결된 페이지 헤더 제목 변경
- 각 페이지 파일의 `render_page_header()` 함수 호출 부분 수정
- 예: `ui_pages/daily_input_hub.py`: `render_page_header("일일 마감", "📝")` → `render_page_header("오늘 마감 입력", "📝")`
- 예: `ui_pages/sales_entry.py`: `render_page_header("🛠 매출 보정 / 과거 매출 입력", "💰")` → `render_page_header("매출/방문자 입력", "💰")`
- 예: `ui_pages/health_check/health_check_page.py`: `render_page_header("매장 체크리스트", "📋")` → `render_page_header("QSC 입력", "📋")`
- 예: `ui_pages/settlement_actual.py`: 헤더가 없으면 추가 필요 → `render_page_header("월간 정산 입력", "📅")`
- 예: `ui_pages/target_sales_structure.py`: `render_page_header("목표 매출 구조 입력", "📈")` → `render_page_header("매출 목표 입력", "📈")`
- 예: `ui_pages/target_cost_structure.py`: `render_page_header("목표 비용 구조(입력)", "💳")` → `render_page_header("비용 목표 입력", "💳")`
- 예: `ui_pages/input/menu_input.py`: `render_page_header("📘 판매 메뉴 입력", "📘")` → `render_page_header("메뉴 입력", "📘")`
- 예: `ui_pages/input/ingredient_input.py`: `render_page_header("🧺 사용 재료 입력", "🧺")` → `render_page_header("재료 입력", "🧺")`

### 6.6 동적 라벨 처리
- 누락 건수나 상태 표시가 있는 버튼의 경우, "입력" 앞에 정보 추가
  - 예: `"📘 메뉴 입력" + (f" ({assets.get('missing_price')}건)" if assets.get('missing_price', 0) > 0 else "")`
  - 예: `"🎯 매출 목표 입력" + (" (필수)" if not assets.get('has_target') else "")`

### 6.7 가이드 카드 텍스트
- 가이드 카드의 텍스트도 입력 중심으로 조정 가능
- 예: "정교한 분석의 시작은 정확한 데이터 입력입니다." → 유지 (이미 입력 중심)
