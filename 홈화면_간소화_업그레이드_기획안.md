# 홈화면 간소화 업그레이드 기획안

## 🎯 목표
테스트 페이지의 **최대 간결형 디자인**을 실제 홈 페이지에 적용하여, 브랜드 정체성을 강조하고 사용자의 첫 행동을 명확하게 제시하는 간소한 홈 화면으로 전환

---

## 📊 현재 상태 vs 목표 상태

### 현재 홈 페이지 구조 (`home_page_v0.py`)
```
1. 브랜드 히어로
   - CAUSE OS
   - 슬로건
   - [오늘 숫자 입력하기] CTA 버튼

2. 오늘 당신의 매장은 (상태 섹션)
   - 마감 / 레시피 / 목표 미니 카드 3개
   - 오늘 우리 매장 추천
   - 추천 메시지 + 액션 버튼
   - 최근 7일 코치 기록

3. STEP 1: 입력 (상세 카드)
   - 제목 + 설명 박스 + 설명 텍스트
   - [오늘 입력하기] [데이터 입력센터] 버튼

4. STEP 2: 분석 (상세 카드)
   - 제목 + 설명 박스 + 설명 텍스트
   - [데이터 분석센터] 버튼

5. STEP 3: 설계 (상세 카드)
   - 제목 + 설명 박스 + 설명 텍스트
   - [가게 전략 센터] 버튼
```

### 목표 홈 페이지 구조 (테스트 페이지 기반)
```
1. 브랜드 히어로
   - CAUSE OS
   - 슬로건
   - (CTA 버튼 제거)

2. 3단계 운영 흐름 (제목)

3. STEP 1-3 (간결형 카드 3개)
   - 아이콘 + 제목 + 짧은 설명
   - 각 카드 아래 버튼
```

## 🔄 주요 변경 사항

### [제거 대상]
1. ✅ **"오늘 당신의 매장은" 상태 섹션 전체**
   - 마감/레시피/목표 미니 카드
   - 추천 엔진 호출 및 표시
   - 추천 액션 버튼
   - 최근 7일 코치 기록

2. ✅ **브랜드 히어로의 CTA 버튼**
   - "오늘 숫자 입력하기" 버튼 제거

3. ✅ **STEP 카드의 상세 설명**
   - `ps-step-highlight-box` (강조 박스)
   - `ps-step-description` (상세 설명 텍스트)
   - 긴 제목 텍스트

### [유지/변경 대상]
1. ✅ **브랜드 히어로** (CTA 버튼만 제거)
   - CAUSE OS
   - 슬로건
   - 보조 정의

2. ✅ **STEP 구조** (간결형으로 변경)
   - STEP 1, 2, 3 카드 유지
   - 상세 설명 제거, 간결한 형태로 변경
   - 각 카드 아래 버튼 유지

3. ✅ **CSS 스타일** (테스트 페이지 스타일 적용)
   - `.ps-step-guide-compact`
   - `.ps-step-button` (간결형)
   - 간격 조정 (히어로-제목-박스 간격 동일하게)

## 🎨 디자인 원칙

### 간소화 원칙
1. **정보 밀도 감소**
   - 상태 요약 제거
   - 추천 메시지 제거
   - STEP 상세 설명 제거

2. **시각적 위계 명확화**
   - 브랜드 히어로 (최상단)
   - 3단계 운영 흐름 제목
   - STEP 카드 3개 (동일한 위계)

3. **간격 일관성**
   - 히어로 ↔ 제목: 2rem
   - 제목 ↔ STEP 박스: 2rem
   - STEP 박스 ↔ 버튼: 1.5rem

4. **브랜드 중심**
   - 첫 화면에서 브랜드 정체성 강조
   - 기능 설명보다 브랜드 메시지 우선

## ⚠️ 주의사항

### 유지해야 할 기능
1. **버튼 라우팅**
   - STEP 버튼들의 페이지 이동 기능 유지
   - `st.session_state.current_page` 설정 유지

2. **CSS 주입**
   - `inject_home_premium_css()` 호출 유지

3. **에러 처리**
