## 1. 입력 허브 개선안

### 1.1 현재 구조의 문제점

1. **정보 과부하**: 5개 섹션에 11개 버튼이 한 화면에 노출되어 선택이 어려움
2. **우선순위 불명확**: 자주 쓰는 것과 가끔 쓰는 것이 동일한 비중으로 표시
3. **컨텍스트 부족**: 각 입력이 언제, 왜 필요한지 설명이 부족
4. **상태 피드백 부족**: 어떤 입력이 완료되었는지, 다음에 무엇을 해야 하는지 불명확

### 1.2 개선안: 3단계 계층 구조

#### **A. 최상단: 스마트 추천 카드 (강화)**

**현재**: 단순 추천 1개  
**개선**: 상황별 다중 추천 + 진행률 표시

```
┌─────────────────────────────────────────┐
│ 🎯 오늘 해야 할 일                      │
│                                         │
│ ✅ 매출 입력 완료                       │
│ ⏳ 마감 대기 중 (오후 10시 이후 가능)   │
│ 📋 체크리스트 (3일 남음)                │
│                                         │
│ [일일 마감 계속하기] [마감 완료하기]    │
└─────────────────────────────────────────┘
```

**기능**:
- 오늘의 필수 작업 3-5개를 우선순위로 표시
- 각 작업의 완료 상태와 다음 액션 제시
- 진행률 바로 시각적 피드백
- "오늘 입력"과 "마감"은 같은 페이지(일일 마감)로 통합 표시

#### **B. 중간: 빠른 입력 (Quick Actions)**

**현재**: 모든 입력이 동일한 비중  
**개선**: 사용 빈도 기반 2단 구조

```
┌─────────────────────────────────────────┐
│ ⚡ 빠른 입력 (매일 사용)                │
│                                         │
│ [📝 일일 마감]                          │
│                                         │
│ 💡 매일 매출, 네이버 방문자, 판매량을  │
│    입력하고 마감하는 페이지입니다       │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 📅 정기 입력 (주/월 단위)              │
│                                         │
│ [🩺 매장 체크리스트] [📅 월간 정산]    │
│                                         │
│ 💡 주 1-2회 체크리스트, 월 1회 정산     │
└─────────────────────────────────────────┘
```

**개선점**:
- 빠른 입력에는 일일 마감만 (오늘 입력과 마감은 같은 페이지)
- 정기 입력에 체크리스트 추가 (주간 리포트 제거)
- 각 버튼에 "마지막 입력일" 표시 (예: "3일 전 입력함")
- 완료 상태 배지 표시

#### **C. 하단: 설정 입력 (Setup & Correction)**

**현재**: 매장 자산 입력이 너무 강조됨  
**개선**: 접을 수 있는 섹션으로 재구성

```
┌─────────────────────────────────────────┐
│ 🏗️ 매장 설정 (처음 한 번만)            │
│                                         │
│ [메뉴] [재료] [레시피]                 │
│                                         │
│ 📊 완성도: 메뉴 12개 ✓ | 재료 8개 ✓   │
│           레시피 5개 ⚠ (부족)         │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 🎯 목표 설정 (월 1회)                  │
│                                         │
│ [목표 매출 구조] [목표 비용 구조]       │
│                                         │
│ 💡 설계 페이지에서 사용하는 기준 데이터 │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 🔧 보정 도구 (필요할 때만)             │
│                                         │
│ [매출 보정] [판매량 보정]              │
│                                         │
│ ⚠️ 일반 입력은 "오늘 입력"을 사용하세요 │
└─────────────────────────────────────────┘
```

**개선점**:
- 각 섹션을 expander로 접을 수 있게 (기본 접힘)
- 완성도/상태 표시로 다음 액션 유도
- 보정 도구는 경고 스타일로 표시

### 1.3 입력 허브 개선안 요약

| 영역 | 현재 | 개선안 |
|------|------|--------|
| **최상단** | 단순 추천 1개 | 다중 추천 + 진행률 + 상태 표시 |
| **빠른 입력** | 모든 버튼 동일 | 일일 마감만 (오늘 입력=마감 통합) |
| **정기 입력** | 주간 리포트 포함 | 체크리스트 + 월간 정산 (주간 리포트 제거) |
| **설정 입력** | 평면적 나열 | expander로 접기 + 완성도 표시 |
| **시각적 계층** | 약함 | 명확한 3단계 계층 (필수 → 정기 → 설정) |

## 2. 일일 마감 개선안

**참고**: "오늘 입력"과 "마감"은 실제로 같은 페이지입니다. 통합하여 "일일 마감"으로 명명합니다.

### 2.1 현재 구조의 문제점

1. **입력 흐름이 비직관적**: 매출 → 네이버 방문자 → 판매량 → 메모 순서가 자연스럽지 않음
2. **상태 표시가 약함**: "임시 기록" vs "마감 완료"의 차이가 명확하지 않음
3. **저장/마감 구분이 애매함**: 언제 저장하고 언제 마감하는지 불명확
4. **피드백 부족**: 저장 후 다음에 무엇을 해야 하는지 안내가 약함
5. **용어 혼란**: "오늘 입력"과 "마감"이 실제로는 같은 페이지인데 구분되어 보임

### 2.2 개선안: 단계별 입력 플로우

#### **A. 상단: 날짜 & 상태 대시보드**

```
┌─────────────────────────────────────────┐
│ 📅 2026년 1월 24일 (목요일)            │
│                                         │
│ 상태: ⚠️ 임시 기록 중                   │
│                                         │
│ [매출] [네이버 방문자] [판매량] [메모] │
│  ✓         ✓            ⚠       -      │
│                                         │
│ 💰 총 매출: 450,000원                  │
│ 👥 네이버 방문자: 25명                 │
│ 📦 판매 메뉴: 3개                      │
└─────────────────────────────────────────┘
```

**개선점**:
- 각 입력 항목의 완료 상태를 아이콘으로 표시
- 현재까지 입력된 값 요약을 한눈에 표시
- 상태에 따라 다음 액션 제시

#### **B. 입력 영역: 탭 기반 단계별 입력**

**현재**: 모든 입력이 한 화면에 섞여 있음  
**개선**: 탭으로 단계 구분

```
[💰 매출] [👥 네이버 방문자] [📦 판매량] [📝 메모]

┌─────────────────────────────────────────┐
│ 💰 매출 입력                            │
│                                         │
│ 카드 매출: [______] 원                  │
│ 현금 매출: [______] 원                  │
│                                         │
│ 총 매출: 0원 (자동 계산)                │
│                                         │
│ 💡 팁: 카드/현금 중 하나만 입력해도 됨  │
│                                         │
│ [임시 저장] [다음: 네이버 방문자 입력]  │
└─────────────────────────────────────────┘
```

**탭 구조**:
1. **💰 매출**: 카드/현금 입력 (필수 권장)
2. **👥 네이버 방문자**: 네이버 스마트플레이스 방문자 수 (선택)
3. **📦 판매량**: 메뉴별 판매량 (선택, 메뉴가 있을 때만)
4. **📝 메모**: 운영 메모 (선택)

**개선점**:
- 각 탭에서 해당 항목만 집중
- 탭 간 이동으로 자연스러운 입력 흐름
- 각 탭에서 "임시 저장" 가능
- 마지막 탭에서만 "마감하기" 버튼 표시

#### **C. 하단: 액션 버튼 영역**

```
┌─────────────────────────────────────────┐
│ 💾 저장 옵션                            │
│                                         │
│ [💾 임시 저장] (지금까지 입력 저장)     │
│                                         │
│ [📋 마감하기] (오늘 입력 완료)          │
│    ⚠️ 마감 후에는 보정만 가능합니다     │
└─────────────────────────────────────────┘
```

**개선점**:
- "임시 저장"과 "마감하기"를 명확히 구분
- 마감하기는 모든 탭을 완료한 후에만 활성화
- 마감 전 확인 다이얼로그 표시

### 2.3 일일 마감 개선안 요약

| 영역 | 현재 | 개선안 |
|------|------|--------|
| **상단 상태** | 단순 배지 | 대시보드 형태 + 각 항목 완료 상태 |
| **입력 방식** | 한 화면에 모든 입력 | 탭 기반 단계별 입력 |
| **저장/마감** | 구분이 애매 | 명확한 2단계 (임시 저장 vs 마감) |
| **피드백** | 약함 | 각 단계별 완료 상태 + 다음 액션 제시 |

## 3. 매장 자산 입력 페이지 개선안

### 3.1 메뉴 입력 개선안

#### **현재 문제점**
- 단일/일괄 입력 탭이 있지만, 어떤 것을 써야 할지 불명확
- 등록된 메뉴 목록이 단순 테이블로만 표시
- 메뉴 간 관계나 카테고리 정보가 없음

#### **개선안: 통합 입력 인터페이스**

```
┌─────────────────────────────────────────┐
│ 📘 메뉴 입력                             │
│                                         │
│ [➕ 새 메뉴 추가] [📋 일괄 등록]        │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ ➕ 새 메뉴 추가                          │
│                                         │
│ 메뉴명: [________________]               │
│ 판매가: [______] 원                     │
│ 카테고리: [선택 ▼] (선택사항)           │
│                                         │
│ [💾 저장] [➕ 추가 후 계속 입력]         │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 📋 등록된 메뉴 (12개)                   │
│                                         │
│ [검색: ______] [카테고리 필터 ▼]        │
│                                         │
│ ┌─────────┬─────────┬─────────┐        │
│ │ 메뉴명   │ 판매가  │ 상태    │        │
│ ├─────────┼─────────┼─────────┤        │
│ │ 파스타   │ 15,000  │ ✓ 레시피│        │
│ │ 스테이크 │ 25,000  │ ⚠ 레시피│        │
│ └─────────┴─────────┴─────────┘        │
│                                         │
│ 💡 레시피가 없으면 원가 계산 불가       │
│ [레시피 입력하러 가기]                  │
└─────────────────────────────────────────┘
```

**개선점**:
- 단일/일괄 입력을 상단 버튼으로 선택
- 등록된 메뉴 목록에 상태 표시 (레시피 있음/없음)
- 검색/필터 기능 추가
- 다음 액션 제시 (레시피 입력 유도)

### 3.2 재료 입력 개선안

#### **현재 문제점**
- 단일 입력만 가능, 일괄 입력이 없음
- 재료 간 관계(대체재, 관련재료) 정보 없음
- 단위 변환이 복잡함

#### **개선안: 스마트 재료 입력**

```
┌─────────────────────────────────────────┐
│ 🧺 재료 입력                             │
│                                         │
│ [➕ 새 재료 추가] [📋 일괄 등록]        │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ ➕ 새 재료 추가                          │
│                                         │
│ 재료명: [________________]               │
│                                         │
│ 단위 설정:                              │
│   기준 단위: [kg ▼]                     │
│   단가: [______] 원 / kg                │
│                                         │
│ 발주 단위 (선택):                       │
│   발주 단위: [박스 ▼]                   │
│   변환율: 1박스 = [__] kg               │
│                                         │
│ [💾 저장]                                │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 📋 등록된 재료 (8개)                     │
│                                         │
│ [검색: ______]                          │
│                                         │
│ ┌─────────┬─────────┬─────────┐        │
│ │ 재료명   │ 단위    │ 단가    │        │
│ ├─────────┼─────────┼─────────┤        │
│ │ 밀가루   │ kg      │ 3,000   │        │
│ │ 토마토   │ kg      │ 5,000   │        │
│ └─────────┴─────────┴─────────┘        │
│                                         │
│ 💡 재료를 등록하면 레시피 입력이 쉬워집니다 │
└─────────────────────────────────────────┘
```

**개선점**:
- 일괄 등록 기능 추가
- 단위 설정을 단계별로 안내
- 발주 단위 변환을 자동 계산
- 검색 기능 추가

### 3.3 레시피 입력 개선안

#### **현재 문제점**
- 일괄 입력만 가능, 단일 입력이 불편함
- 메뉴별 레시피 관리가 어려움
- 원가 계산 결과가 실시간으로 보이지 않음

#### **개선안: 메뉴 중심 레시피 관리**

```
┌─────────────────────────────────────────┐
│ 🧑‍🍳 레시피 입력                         │
│                                         │
│ 메뉴 선택: [파스타 ▼]                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 📝 파스타 레시피                         │
│                                         │
│ 재료 추가:                              │
│   [재료 선택 ▼] [사용량: __] [단위: g] │
│   [➕ 추가]                              │
│                                         │
│ 등록된 재료:                            │
│   • 밀가루: 200g (600원)                │
│   • 토마토: 150g (750원)                │
│   • 올리브오일: 30ml (450원)            │
│                                         │
│ ┌─────────────────────────────────┐    │
│ │ 💰 원가 계산                      │    │
│ │   재료비 합계: 1,800원           │    │
│ │   판매가: 15,000원               │    │
│ │   원가율: 12.0% ✓ (양호)         │    │
│ └─────────────────────────────────┘    │
│                                         │
│ 조리방법:                               │
│   [_____________________________]       │
│   [_____________________________]       │
│                                         │
│ [💾 저장] [다음 메뉴]                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 📋 레시피 현황                           │
│                                         │
│ 전체 메뉴: 12개                         │
│ 레시피 등록: 8개 (67%)                  │
│                                         │
│ [레시피 없는 메뉴 보기]                 │
└─────────────────────────────────────────┘
```

**개선점**:
- 메뉴 선택 후 해당 메뉴의 레시피만 집중
- 재료 추가를 동적으로 (➕ 버튼으로 계속 추가)
- 원가 계산 결과를 실시간 표시
- 레시피 등록률을 시각적으로 표시

### 3.4 매장 자산 입력 개선안 요약

| 페이지 | 현재 | 개선안 |
|--------|------|--------|

## 6. 전체 개선안 요약

### 6.1 입력 허브

| 개선 항목 | 내용 |
|-----------|------|
| **계층 구조** | 3단계 (필수 → 정기 → 설정) |
| **스마트 추천** | 다중 추천 + 진행률 + 상태 표시 |
| **빠른 입력** | 사용 빈도별 2단 구조 |
| **설정 입력** | Expander로 접기 + 완성도 표시 |

### 6.2 일일 마감

| 개선 항목 | 내용 |
|-----------|------|
| **페이지 통합** | "오늘 입력"과 "마감"을 하나의 페이지로 통합 |
| **입력 방식** | 탭 기반 단계별 입력 |
| **상태 표시** | 대시보드 형태 + 각 항목 완료 상태 |
| **저장/마감** | 명확한 2단계 구분 (임시 저장 vs 마감 완료) |
| **용어 통일** | "방문자" → "네이버 방문자"로 명확화 |

### 6.3 매장 자산 입력

| 개선 항목 | 내용 |
|-----------|------|
| **메뉴 입력** | 통합 인터페이스 + 상태 표시 + 검색 |
| **재료 입력** | 일괄 등록 추가 + 단위 설정 안내 |
| **레시피 입력** | 메뉴 중심 관리 + 실시간 원가 계산 |

### 6.4 운영 입력

| 개선 항목 | 내용 |
|-----------|------|
| **체크리스트** | 정기 입력 섹션에 포함, 바로 시작 가능 |
| **월간 정산** | 상태 확인 + 바로가기 |
| **주간 리포트** | 제거됨 (앱에서 완전 제거) |

### 6.5 보정 입력

| 개선 항목 | 내용 |
|-----------|------|
| **명확한 구분** | 보정 vs 일반 입력 구분 |
| **기존 값 표시** | 보정 전 값과 비교 |
| **사유 입력** | 보정 이유 기록 (선택) |

## 7. 구현 우선순위

### Phase 1: 핵심 개선 (즉시)
1. ✅ 입력 허브: 스마트 추천 강화 + 빠른 입력/정기 입력 구조 정리
2. ✅ 일일 마감: 탭 기반 단계별 입력 (오늘 입력=마감 통합)
3. ✅ 매장 자산 입력: 상태 표시 추가
4. ✅ 주간 리포트: 입력 허브 및 앱에서 완전 제거
5. ✅ 용어 통일: "방문자" → "네이버 방문자"로 전체 변경

### Phase 2: UX 개선 (단기)
1. 메뉴/재료 입력: 검색/필터 기능
2. 레시피 입력: 메뉴 중심 관리
3. 일일 마감: 대시보드 강화

### Phase 3: 고도화 (중기)
1. 입력 허브: 완성도 표시
2. 보정 입력: 보정 사유 입력
3. 전체 입력 플로우 최적화
