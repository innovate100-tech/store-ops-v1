## 2️⃣ 페이지별 기능 요약표

### 🏠 홈

#### [홈 / home_page.py]
- **목적**: 사장 계기판 - 통합 상태 + 업무 지시
- **사장이 하는 행동**: 오늘의 운영 지시 확인, 전략 요약 확인, 문제 인식, 상태판 확인
- **입력하는 데이터**: 없음 (조회 전용)
- **보여주는 분석/결과**:
  - 미니 코칭 카드 (한 줄 코칭 + 오늘 할 일 1개)
  - ZONE 0: 오늘의 운영 지시 (1개 행동)
  - ZONE 1: 이번 달 가게 전략 요약 (전략 카드 TOP3)
  - ZONE 2: 문제 인식 & 빠른 진입
  - ZONE 3: 오늘 상태판 (KPI: 월매출, 어제매출, 마감률, 유입당매출, 월이익, 목표대비)
  - ZONE 4: 이번 주 우선순위 TOP3 (lazy)
  - ZONE 5: 가게 구조 스냅샷 (lazy)
- **설계 요소 포함 여부**: 없음 (조회 전용)

### ✍ 입력

#### [일일 입력(통합) / daily_input_hub.py]
- **목적**: 일일 데이터 통합 입력 (매출, 방문자, 판매량, 메모)
- **사장이 하는 행동**: 날짜 선택, 매출 입력(카드/현금), 네이버 방문자 입력, 판매량 입력(메뉴별), 메모 입력, 저장/마감
- **입력하는 데이터**:
  - 매출 (카드/현금/총매출)
  - 네이버 방문자 수
  - 판매량 (메뉴별)
  - 메모
- **보여주는 분석/결과**: 날짜 상태 (마감 완료/임시 기록/미입력), 현재 저장된 값 요약
- **설계 요소 포함 여부**: 없음 (입력 전용)

- **사장이 하는 행동**: 날짜 선택, 매출 입력, 방문자 입력, 판매량 입력(메뉴별), 이슈 체크(품절/컴플레인/단체손님/직원이슈), 메모 입력, 마감 완료

  - 판매량 (메뉴별)
  - 이슈 (품절/컴플레인/단체손님/직원이슈)

## 5️⃣ 설계 영역 요약 문장

**"이 앱의 매장 설계는 현재 다음 5개 영역으로 구성돼 있다:"**

1. **메뉴 포트폴리오 설계**: 메뉴 등록 및 역할(주력/보조/유인/기타)/카테고리(메인/사이드/주류/디저트) 설정으로 포트폴리오 균형 조정
2. **메뉴 수익 구조 설계**: 메뉴별 판매가 조정으로 원가율 최적화 (원가율 목표 달성)
3. **재료 구조 설계**: 재료 등록 및 단가 설정으로 재료 구조 최적화 (TOP3 집중도 관리, 고위험 재료 식별)
4. **원가 구조 설계**: 레시피 수정(메뉴-재료 연결, 사용량 조정)으로 메뉴별 원가 계산 및 원가율 최적화
5. **수익 구조 설계**: 비용 구조 입력(고정비/변동비 항목) 및 조정으로 손익분기점 최적화

**설계 진입점**: 가게 설계 센터 (통합 진단실) - 4개 설계 영역 상태 요약, 판결문, 추천 액션

**설계 결과물**: 
- 포트폴리오 균형 점수 (0-100), 균형 상태 (균형/주의/위험)
- 메뉴별 원가, 원가율, 공헌이익
- 재료 TOP3 집중도, 고위험 재료 목록
- 손익분기점, 고정비/변동비 비율, 손익분기점 대비 매출 비율

**설계 자동화 (전략 생성)**: 
- 가게 상태 분류 (건강/주의/위험)
- 전략 카드 생성 (TOP3)
- 실행 로드맵 생성 (이번 주 TOP3)
- 근거 생성 (숫자+설계+검진)
- 판결문 생성 (가장 의심되는 구조)

**E. 목표 매출 입력** (별도 입력 경로)
- 입력: 목표 매출, 목표 원가율, 목표 인건비율, 목표 임대료율, 목표 기타비용율, 목표 순이익률
- 설계 입력: 매출 구조 목표
- 저장 위치: `targets` 테이블

#### 2단계: 설계 변경

**A. 메뉴 포트폴리오 설계**
- 변경: 메뉴 역할 변경, 메뉴 카테고리 변경
- 평가: 포트폴리오 균형 점수 계산 (`calculate_portfolio_balance_score`)
- 관련 파일: `ui_pages/design_lab/menu_portfolio_helpers.py`

**B. 메뉴 수익 구조 설계**
- 변경: 메뉴 판매가 수정
- 평가: 원가율 계산, 공헌이익 계산, 고원가율 메뉴 개수
- 관련 파일: `ui_pages/menu_profit_design_lab.py`, `src/analytics.py`

**C. 재료 구조 설계**
- 변경: 재료 단가 수정, 재료 추가/삭제
- 평가: TOP3 집중도 계산 (`calculate_cost_concentration`), 고위험 재료 식별 (`identify_high_risk_ingredients`)
- 관련 파일: `ui_pages/design_lab/ingredient_structure_helpers.py`

**D. 원가 구조 설계**
- 변경: 레시피 수정 (재료 사용량 조정), 레시피 추가/삭제
- 평가: 메뉴별 원가 계산 (`calculate_menu_cost`), 원가율 계산
- 관련 파일: `app.py` (레시피 관리), `src/analytics.py`

**E. 수익 구조 설계**
- 변경: 비용 항목 추가/수정/삭제, 고정비/변동비 구조 조정
- 평가: 손익분기점 계산 (`calculate_break_even_sales`), 고정비/변동비 비율 계산
- 관련 파일: `ui_pages/target_cost_structure.py`, `ui_pages/revenue_structure_design_lab.py`

#### 3단계: 설계 평가 (자동 계산)

**A. 메뉴 포트폴리오 평가**
- 함수: `calculate_portfolio_balance_score()` (`ui_pages/design_lab/menu_portfolio_helpers.py`)
- 출력: 균형 점수 (0-100), 균형 상태 (균형/주의/위험), 미분류 비율

**B. 메뉴 수익 평가**
- 함수: `calculate_menu_cost()` (`src/analytics.py`)
- 출력: 메뉴별 원가, 원가율, 공헌이익, 고원가율 메뉴 개수

**C. 재료 구조 평가**
- 함수: `calculate_cost_concentration()`, `identify_high_risk_ingredients()` (`ui_pages/design_lab/ingredient_structure_helpers.py`)
- 출력: TOP3 집중도, 고위험 재료 목록, 발주 구조 상태

**D. 수익 구조 평가**
- 함수: `calculate_break_even_sales()`, `get_fixed_costs()`, `get_variable_cost_ratio()` (`src/storage_supabase.py`)
- 출력: 손익분기점, 고정비, 변동비율, 손익분기점 대비 매출 비율

#### 4단계: 전략 생성 (자동화)

**A. 가게 상태 분류**
- 함수: `classify_store_state()` (`ui_pages/strategy/store_state.py`)
- 입력: 매출 데이터, 설계 상태, 월별 데이터
- 출력: 가게 상태 (survival/recovery/restructure/growth), 점수, 신호, 근거

**B. 전략 카드 생성**
- 함수: `build_strategy_cards()` (`ui_pages/strategy/strategy_cards.py`)
- 입력: 가게 상태 분류 결과, 설계 인사이트, 건강검진 결과
- 출력: 전략 카드 TOP3 (제목, 이유, 영향, 실행 계획, CTA)

**C. 실행 로드맵 생성**
- 함수: `build_weekly_roadmap()` (`ui_pages/strategy/roadmap.py`)
- 입력: 전략 카드 결과
- 출력: 이번 주 실행 TOP3 (작업, 예상 시간, CTA)

**D. 근거 생성**
- 함수: `build_evidence_bundle()` (`ui_pages/strategy/evidence_builder.py`)
- 입력: 매출 데이터, 설계 인사이트, 건강검진 결과
- 출력: 근거 번들 (숫자 근거, 설계 근거, 검진 근거)

**E. 판결문 생성**
- 함수: `get_design_center_verdict()`, `get_primary_concern()` (`ui_pages/coach/coach_adapters.py`, `ui_pages/design_lab/design_center_data.py`)
- 입력: 설계 센터 요약 데이터
- 출력: 판결문, 추천 액션
