# 입력·분석·설계 — 필요 페이지 정리

**작성 일자**: 2026-01-24  
**목적**: 입력·분석·설계 기준으로, **실제로 필요한 페이지**만 정리

---

## 📋 개요

입력은 틀을 잡은 상태입니다.  
아래는 **분석 쪽에서 추가로 필요한 페이지**와 **이미 있는 페이지**를 구분한 정리입니다.

---

## 🔍 분석 페이지 현황 및 필요 사항

### 1. 목표 매출구조

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | 목표 매출 구조 입력 | ✅ 완료 |
| **분석** | 매출 분석 | ✅ **기존 매출 관리(매출 분석) 페이지로 충족** |

- 목표 매출구조 전용 분석 페이지 **추가 불필요**
- 매출 분석 페이지(`매출 관리`)에서 목표·실적 등 매출 분석 처리

---

### 2. 목표 비용구조

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | 목표 비용 구조(입력) | ✅ 완료 |
| **분석** | **비용 분석** | ❌ **필요** (신규) |

- **비용 분석 페이지**를 새로 만들어야 함
- 담당 기능 (예시):
  - 목표매출 달성 시 비용구조 분석
  - 월간 집계
  - 매출 수준별 비용·영업이익 시뮬레이션
  - 고정비/변동비 비율, 손익분기점 등
- 위치: 🟣 장사 분석 카테고리

---

### 3. 실제정산 (월간 성적표)

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | 실제정산 (실제 비용·매출 입력) | ✅ 있음 (입력·분석 혼재) |
| **분석** | **실제정산 분석** | ❌ **필요** (신규) |

- **실제정산 분석 페이지** 필요
- 담당 기능 (예시):
  - 목표 vs 실제 비교
  - 월간 성적표 상세
  - 월별 트렌드, 카테고리별 비용·수익성 분석
- 위치: 🔴 성적표 카테고리  
- 추후: 실제정산 페이지를 **입력 전용**으로 정리하고, 분석은 이 페이지로 이전하는 방안 검토

---

### 4. 재고 (발주 관리 → 재고로 전환)

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | 재고 입력 | ✅ 완료 |
| **분석** | 재고 분석 | ✅ **이미 있음** (`ui_pages/analysis/inventory_analysis.py`) |

- 발주 관리 페이지는 **제거된 상태**
- 재고 입력 ↔ **재고 분석** 페이지로 대응 구조 완료
- 추가 작업 불필요

---

### 5. 재료 사용량 집계

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | (없음, 판매량 입력 시 자동 계산) | - |
| **분석** | 재료 사용량 집계 | ✅ **필요** (기존 페이지 유지) |

- **재료 사용량 집계** 페이지 필요 → 현재 분석 전용 페이지 유지
- 기간별 사용량, ABC 분석, 재료별 집계 등
- 위치: 🔵 내일 준비 카테고리 (현행 유지)

---

### 6. 메뉴 / 재료 / 레시피

| 구분 | 페이지 | 상태 |
|------|--------|------|
| **입력** | 판매 메뉴, 사용 재료, 판매 레시피 입력 | ✅ 완료 |
| **설계** | 메뉴 포트폴리오 설계실, 재료 구조 설계실 등 | ✅ 완료 |

- 입력–설계 분리 완료, 추가 필요 페이지 없음

---

## 📊 필요 페이지 요약

| 항목 | 입력 | 분석 | 비고 |
|------|------|------|------|
| **목표 매출구조** | ✅ | ✅ 매출 분석으로 충족 | 별도 분석 페이지 불필요 |
| **목표 비용구조** | ✅ | ❌ **비용 분석 페이지 필요** | 신규 |
| **실제정산** | ✅ | ❌ **실제정산 분석 페이지 필요** | 신규 |
| **재고** | ✅ | ✅ 재고 분석 있음 | 발주 제거, 재고로 통합 완료 |
| **재료 사용량** | - | ✅ 재료 사용량 집계 유지 | 필요 |
| **메뉴/재료/레시피** | ✅ | - | 설계실과 분리 완료 |

## 🎯 신규로 필요한 분석 페이지 (2개)

### 1. 비용 분석 페이지

- **이름(가칭)**: 비용 분석
- **위치**: 🟣 장사 분석
- **역할**: 목표 비용구조 입력 데이터 기반 비용 분석
- **기능 예시**:
  - 목표매출 달성 시 비용구조 분석
  - 월간 집계, 고정비/변동비 비율
  - 매출 수준별 비용·영업이익 시뮬레이션
  - 손익분기점 시각화 등
- **구현**: `ui_pages/analysis/cost_analysis.py` 등 신규 생성

### 2. 실제정산 분석 페이지

- **이름(가칭)**: 실제정산 분석 (또는 월간 성적표 분석)
- **위치**: 🔴 성적표
- **역할**: 실제정산 입력 데이터 기반 성과 분석
- **기능 예시**:
  - 목표 vs 실제 비교
  - 월간 성적표 상세, 월별 트렌드
  - 카테고리별 비용·수익성 분석, 차트 등
- **구현**: `ui_pages/analysis/settlement_analysis.py` 등 신규 생성

## ✅ 정리

- **이미 있는 분석 페이지**
  - 매출 분석(매출 관리), 재고 분석, 재료 사용량 집계, 원가 파악 등
- **추가로 필요한 분석 페이지**
  - **비용 분석** (목표 비용구조 대응)
