# 비용구조 기능 구현 계획

## 메뉴 구조 변경

### 기존 구조
- 매출
- 비용  
- 기타

### 새로운 구조
- **매출**: 점장 마감, 매출 관리, 판매 관리, 발주 관리
- **비용**: 메뉴 관리, 재료 관리, 레시피 관리, 원가 분석
- **재무**: 비용구조 (신규)
- **기타**: 주간 리포트, 통합 대시보드

## 비용구조 기능

### 5개 비용 카테고리
1. **임차료** - 임대료 관련 모든 비용
2. **재료비** - 원재료 구매 비용
3. **인건비** - 직원 급여 및 복리후생
4. **공과금** - 전기, 가스, 수도 등
5. **부가세&카드수수료** - VAT, 카드사 수수료 등

### 데이터 입력 방식
- **월간 1회 입력**: 각 월마다 한 번만 입력 가능
- **세부 항목 입력**: 각 카테고리별로 여러 항목 입력 가능
  - 예: 인건비 → "메인 요리사", "서빙 직원" 등 개별 입력
- **집계**: 월별로 각 카테고리별 총액 자동 집계

### DB 스키마
```sql
CREATE TABLE expense_structure (
    id UUID PRIMARY KEY,
    store_id UUID,
    year INTEGER,
    month INTEGER,
    category TEXT,  -- '임차료', '재료비', '인건비', '공과금', '부가세&카드수수료'
    item_name TEXT,  -- 세부 항목명
    amount NUMERIC(12, 0),  -- 금액
    notes TEXT,  -- 메모
    created_at TIMESTAMPTZ,
    updated_at TIMESTAMPTZ
);
```

### UI 구성
1. **기간 선택**: 연도/월 선택
2. **5개 카테고리별 입력 폼**: 각 카테고리에 세부 항목 추가
3. **월간 집계 표시**: 카테고리별 총액 + 전체 합계
